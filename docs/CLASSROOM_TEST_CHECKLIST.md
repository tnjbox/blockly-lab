# Blockly Lab 課堂測試檢核表

> 版本：MVP-J11  
> 適用專案：Blockly Lab  
> 適用情境：每次改版後、GitHub Pages 發布後、課堂使用前  
> 建立目的：提供教師在上課前快速確認平台、題庫、評分、成績上傳與 SmartRing 功能是否正常。

---

## 1. 測試前準備

### 1.1 專案狀態檢查

執行指令：

    cd C:\YOSEP\blockly-lab
    git status

確認項目：

- 工作區乾淨，或已確認目前修改屬於本次測試內容。
- 目前分支為 main。
- 本機版本已與 GitHub 同步。

---

### 1.2 套件與建置檢查

若是新電腦或重新下載專案，先執行：

    npm install

或：

    npm ci

建置測試：

    npm run build

確認項目：

- build 成功。
- 沒有 JavaScript 語法錯誤。
- 沒有課程 JS 匯入錯誤。
- 沒有 Vite 打包錯誤。

---

### 1.3 本機啟動測試

開發模式：

    npm run dev

或預覽模式：

    npm run preview

確認項目：

- 網頁可以開啟。
- Blockly 工作區正常顯示。
- 課程代碼輸入區正常顯示。
- 任務選單正常顯示。
- 右側題目面板正常顯示。
- 測試結果區正常顯示。

## 5. 載入範例與成績上傳規則測試

### 5.1 示範題載入範例後不可上傳

測試流程：

1. 載入 JSB00。
2. 選擇 A-01-0。
3. 點選載入範例。
4. 執行系統評分。
5. 檢查上傳成績按鈕。

確認項目：

- 系統評分可執行。
- 評分結果可顯示。
- 上傳成績按鈕不可用。
- 不會寫入 Google Sheet。

---

### 5.2 清除工作區後可重新作答

測試流程：

1. 接續上一項測試。
2. 按清除工作區。
3. 自行拖曳積木完成答案。
4. 執行系統評分。
5. 檢查上傳成績按鈕。

確認項目：

- 清除工作區後，原本載入的範例積木消失。
- sampleLoaded 狀態被重置。
- 學生自行完成後可評分。
- 評分完成後，上傳成績按鈕可用。

---

### 5.3 練習題不可載入範例

測試流程：

1. 載入 JSB00 或 JSA00。
2. 選擇任一 -1 題。

- 載入範例按鈕停用。
- 不會誤載入上一題範例。

## 10. 常見異常檢查

### 10.1 課程載入失敗

可能原因：

- 課程沒有加入 src/courses/index.js。
- 課程 code 拼錯。
- 課程 JS 語法錯誤。
- export default 遺漏。
- npm run build 未通過。

---

### 10.2 載入範例沒有反應

可能原因：

- starterXml 為空。
- starterXml XML 格式錯誤。
- 積木 type 名稱不存在。
- Blockly 版本或積木定義不一致。
- 題目切換時狀態未重置。

---

### 10.3 系統評分錯誤

可能原因：

- testCases 格式錯誤。
- 預期輸出多了空白或換行。
- 學生輸出格式不同。
- 程式產生例外錯誤。
- input 解析邏輯不符合題目需求。

---

### 10.4 上傳成績失敗

可能原因：

- 未填寫學生資料。
- 尚未完成系統評分。
- 已載入範例，不允許上傳。
- Apps Script URL 錯誤。
- Google Sheet 權限錯誤。
- 網路連線異常。

---

### 10.5 SmartRing 無法連線

可能原因：

- 瀏覽器不支援 WebSerial。
- 沒有使用 Chrome 或 Edge。
- COM 埠被 Arduino IDE 或其他程式占用。
- 韌體沒有正確燒錄。
- USB 線材僅支援充電，不支援資料傳輸。
- JSON 協定不一致。
